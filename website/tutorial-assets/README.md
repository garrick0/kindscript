# Tutorial Assets

This directory contains static artifacts used in the interactive tutorial lessons.

## Contents

### `storybook/`
Complete copy of the storybook application from `~/dev/induction-studio/apps/storybook/`

- **Size:** ~65 MB (345 TypeScript/JSON files)
- **Purpose:** Used in Part 4 (Design System - Atoms) and Part 5 (Design System - Molecules) lessons
- **Structure:** Full storybook app with atoms, molecules, organisms, and pages
- **Updated:** 2026-02-12 (manual sync from induction-studio)

### `storybook-files.json`
Flattened JSON representation of all storybook files for easy programmatic access.

- **Size:** 1.9 MB
- **Generated by:** `scripts/load-storybook-files.mjs`
- **Used in:** `src/lib/lessons/storybook-files.ts` helper module
- **Format:** Array of `{ path: string, contents: string }` objects

## Usage in Lessons

Lessons can import storybook files using the helper module:

```typescript
import { getStorybookFiles, getAtomFiles, getMoleculeFiles } from './storybook-files';

// Get all storybook files with a custom prefix
const files = getStorybookFiles('storybook/');

// Or filter by component type
const atomFiles = getAtomFiles();
const moleculeFiles = getMoleculeFiles();
```

See `src/lib/lessons/4-1-atom-source.ts` for an example.

## Updating Storybook Files

When the induction-studio storybook app changes and you need to sync:

```bash
# 1. Copy the latest storybook code
cp -r ~/dev/induction-studio/apps/storybook ./tutorial-assets/storybook

# 2. Regenerate the JSON index
node scripts/load-storybook-files.mjs

# 3. Rebuild the website
npm run build
```

## Why This Approach?

We copy the entire storybook filesystem here instead of dynamically importing it because:

1. **Stability:** Tutorial lessons should be stable and not break when induction-studio code changes
2. **Simplicity:** No cross-repo dependencies or complex build orchestration
3. **Offline:** Tutorial works even if induction-studio repo is unavailable
4. **Performance:** All files are bundled at build time, no runtime file I/O

## TypeScript Configuration

The `storybook/` directory is excluded from TypeScript compilation (see `tsconfig.json`) to avoid
type errors from missing dependencies. Only the JSON file and type declaration are included.
